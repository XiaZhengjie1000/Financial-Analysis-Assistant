Guidelines | Data Upload Specifications

bash:
streamlit run app.py

I. Supported File Formats
Mainstream Format Compatibility
.csv: Comma/tab-delimited text files
.xlsx: Modern Excel spreadsheet files
Unsupported for the time being: legacy .xls Excel files, plain .txt files, .json and other formats.

II. File Encoding Standards
Fully automatic dual encoding adaptive recognition:
Sequential encoding detection: utf-8-sig → gb18030 → gbk → utf-8
No manual encoding switching required. Automatic decoding after upload to avoid Chinese garbled text and reading errors.

III. Delimiter Adaptation Rules
Dual delimiter auto-recognition for CSV files:
Comma , separation
Tab separation
Compatible with non-standard CSV files exported from office software, crawler data and mainstream platforms.

IV. Dataset Structure & Layout Requirements
A standard header row or column is required (the first row or first column must contain field names: e.g. Date, Sales Revenue, Sales Volume, Unit Customer Price).
No strict limits on row or column quantity; compatible with small and large datasets.
Data logic: One column corresponds to one indicator and one row represents a single sample, or vice versa. Cross-table logic optimisation is pending.
Custom field names are available for datasets with disorganised row/column labels.

V. Supported Data Fields & Content Types
Numeric DataIntegers, decimals, financial amounts, sales volumes and indicator values. Fully compatible with statistical calculation, visualisation and interpolation filling.
Text & Categorical DataChinese names, category labels, regions, product types and text codes. Available for X-axis classification display.
Date DataStandard date formats are fully supported for time trend analysis.Tip: Date entries should include complete units (e.g. Year 2026).
Data with Missing ValuesBlank and missing cells are natively supported. The system provides six data cleaning solutions:
Retain raw data
Remove samples with missing values
Trend filling
Mean filling
Median filling
Mode filling

VI. Output & Display Modes
Raw Data: Full-screen interactive data table with scroll browsing.
Statistical Analysis: Full-dimensional descriptive statistics (sample size, mean, standard deviation, extreme values, quartiles, category distribution).
Visualisation: Freely switchable bar charts, line charts and scatter plots; complete trend lines for missing value analysis.
Business Insights: Analytical reports generated by the Deepseek-V1 model.

VII. Limitations & Future Optimisations
Visualisation functions have limited compatibility with cross-tab datasets.
The AI business analysis module currently only displays sample analysis frameworks; formal large language model integration is not yet enabled.
