# 【用户须知｜数据上传规范说明】#
一、支持文件格式
1. 主流格式兼容
- ".sv" 逗号/制表符分隔文本文件
- ".xlsx" Excel 现代表格文件
> 暂不支持：".xls"旧版Excel、".txt"纯文本、".json"等其他格式

 二、文件编码规则
全自动双编码自适应识别：
- 按序读取：utf-8-sig → gb18030 → gbk → utf-8 四种常见编码
> 无需手动切换编码，上传后自动解码，杜绝中文乱码、读取报错。

三、分隔符适配规则
CSV文件自动识别双分隔符：
- 英文逗号 "," 分隔
- 制表符 "Tab" 分隔
> 可正常读取办公软件导出、爬虫数据、平台导出的各类非标CSV。

四、数据表结构&行列要求
1. 必须包含"标准表头行或列"（第一行或第一列为字段名：如日期、销售额、销量、客单价等）
2. 无严格行列数量限制：支持小样本/大样本数据集
3. 行列逻辑：若一列为单一指标字段，则一行为单条样本；若一行为单一指标字段，则一列为单条样本；对于交叉表，逻辑待优化
4. 对行列名称杂乱数据，可自行设置“字段名”

五、数据字段&内容类型支持
1. 数值型数据
整数、小数、财务金额、销量、指标数值等，支持统计计算、全部可视化、插值补全。
2. 文本/分类型数据
中文名称、分类标签、地区、品类、文字编号等，可作为X轴分类展示。
3. 日期类数据
常规日期格式可正常读取展示，可用于时间趋势分析。
Tip：日期数据需要带直接的单位，比如“2026年”
4. 含缺失值数据
原生支持空白/缺失单元格，系统可自动对数据进行清洗，提供不清洗、删除存在空缺的样本、取趋势、取平均数、取中位数、取众数，六大方案供您选择

六、呈现形式说明
1. "原始数据"：以交互式表格全屏展示，支持滑动浏览
2. "统计数据"：输出全维度描述性统计（样本量、均值、标准差、极值、四分位数、类别分布）
3. "可视化呈现"：柱状图 / 折线图 / 散点图 自由切换；缺失值趋势连线完整展示
4. "商业洞察"：Deepseek-V1出具相关分析


待优化：
首要目标：交叉表的数据可视化
次要目标：数据格式拓展，时间趋势以外分析
最次目标：数据文件的自动读取（比如自动屏蔽文件中表格解释部分，选取数据部分？），完成泛用性优化
